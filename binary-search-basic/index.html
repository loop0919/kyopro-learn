Binary Search Basics

二分探索の基礎

Yes / No が一度だけ切り替わる問題に直して、境界を高速に探す。

Python while文 bisect AtCoder

概要

二分探索を「境界を探す」発想として眺めます。

Intuition

1〜100の数当て

1つずつ聞くと、最悪100回。中央で分ける質問なら、候補を一気に減らせます。

  • 質問: 「50以下ですか？」
  • Yesなら候補は 1〜50
  • Noなら候補は 51〜100

1〜100の数当てデモ

候補範囲 1〜100
次の質問 50以下ですか?
質問回数 0
1 100
緑線: 秘密の数字 赤線: 質問する境目

Why Fast?

1回ごとに候補を半分にする

候補数が \(N\) なら、必要な質問回数はだいたい \(\log_2 N\) 回です。

Yes / No

数当てを Yes / No で見てみる

競プロでは「答えそのもの」ではなく、「条件を満たすか？」に直して考えます。

  • \(1 \le x\) は Yes
  • \(3 > x\) は No
  • 境界は Yes と No の切れ目

Pattern

二分探索できる形: Yes / No が一度だけ切り替わる

OK Yes, Yes, Yes, No, No
OK No, No, Yes, Yes
NG Yes, No, Yes, No

中央を見たあと「どちら側を捨てるか」が決まることが必要です。この性質を、競プロでは「単調性」と呼びます。

Chapter Summary

概要のまとめ

直感候補を半分にする質問で、探索範囲を一気に小さくする。
見方値の関係を Yes / No の列として見る。
条件Yes と No が一度だけ切り替わる形が必要。

二分探索は「全部試す」ではなく、「境界を探す」ための考え方です。

実装(while文)

めぐる式二分探索で、境界をバグらせにくく書きます。

Implementation / while

まずテンプレート全体を見る

  • この章では、下の形を部品ごとに分解します
  • ok は Yes 側、ng は No 側として置く
  • 最大の Yes を探すとき、最後に残る ok が答え
最大のYesを探すPython
ok = 0
ng = N + 1

while ng - ok > 1:
    mid = (ok + ng) // 2

    if is_valid(mid):
        ok = mid
    else:
        ng = mid

print(ok)

Implementation / while

境界をバグらせない考え方

  • 最大の Yes を探すなら、左側が Yes、右側が No
  • ok と ng で境界を両側からはさむ
  • ループ中も「境界は必ず ok と ng の間」を保つ

Implementation / while

めぐる式: 不変条件を先に決める

  • is_valid(ok) は常に Yes
  • is_valid(ng) は常に No
  • 初期値は、条件が分かっている番兵として置く
めぐる式二分探索の元投稿
最大のYesを探す準備Python
ok = 0          # Yes と分かっている値
ng = N + 1      # No と分かっている値

# ループ中ずっと守る:
# is_valid(ok) == True
# is_valid(ng) == False

Implementation / while

1ステップでは mid だけ判定する

  • mid = (ok + ng) // 2
  • Yes なら、Yes 側の境界を mid まで進める
  • No なら、No 側の境界を mid まで戻す

Implementation / while

終了条件は ok と ng の距離

  • 間に候補がある間だけループする
  • Yes が左、No が右なら ng - ok > 1
  • 小さい方が No、大きい方が Yes なら ok - ng > 1
左右で差分の向きが変わるPython
# Yes ... Yes, No ... No
ok = 0          # Yes 側
ng = N + 1      # No 側
while ng - ok > 1:
    ...

# No ... No, Yes ... Yes
ng = -1         # No 側
ok = N          # Yes 側
while ok - ng > 1:
    ...

Chapter Summary

実装(while文)のまとめ

Invariantok は常に Yes、ng は常に No。
Updatemid を判定し、Yesなら ok、Noなら ng を動かす。
Finishok と ng が隣り合ったら境界が確定し、答えは ok。

不変条件を決めてから書くと、境界条件を追いやすくなります。

実装(bisectライブラリ)

挿入位置と半開区間で、添え字の意味を整理します。

Implementation / bisect

bisect は値ではなく挿入位置を返す

  • 対象は昇順ソート済みの配列
  • 返るのは 0 から N までの「すき間」の番号
  • その位置で配列を左右に分けて読む

Implementation / bisect

bisect_left(A, x)

  • x 以上になる最小の位置を返す
  • A[:l] は \(x\) 未満
  • A[l:] は \(x\) 以上

Implementation / bisect

bisect_right(A, x)

  • x を超える最小の位置を返す
  • A[:r] は \(x\) 以下
  • A[r:] は \(x\) より大きい

Implementation / bisect

[l, r) が \(x\) と等しい区間

  • l = bisect_left(A, x) は \(x\) 以上の左端
  • r = bisect_right(A, x) は \(x\) を超える左端
  • だから A[l:r] がちょうど \(x\) と等しい部分になる

Implementation / bisect

l と r で個数を読む

l = bisect_left(A, x), r = bisect_right(A, x), N = len(A) とします。

数えたいもの 式 読み方
\(x\) 未満lA[:l]
\(x\) 以下rA[:r]
\(x\) 以上N - lA[l:]
\(x\) より大きいN - rA[r:]
\(x\) と等しいr - lA[l:r]

Chapter Summary

実装(bisectライブラリ)のまとめ

Positionbisect は値ではなく、境界となる挿入位置を返す。
Intervall = bisect_left(A, x), r = bisect_right(A, x) なら A[l:r] が \(x\) と等しい区間。
Countl, r, N から、未満・以下・以上・超過・等しい個数を読む。

添え字を「区間」として読むと、個数計算で混乱しにくくなります。

問題を解く

問題を Yes / No が一度だけ切り替わる判定に落とし込む練習をします。

Thinking Pattern

問題を解く型

1値を仮定する

ABC231 C - Counting 2

Counting 2

AtCoder の問題ページ

問題文

\(N\) 人の生徒がいます。生徒 \(i\) の身長は \(A_i\) です。

\(Q\) 個の質問があり、各質問では整数 \(x_j\) が与えられます。身長が \(x_j\) 以上である生徒の人数を答えます。

制約

  • \(1 \le N,Q \le 2 \times 10^5\)
  • \(1 \le A_i,x_j \le 10^9\)

ABC231 C

愚直に数えるとどうなるか

  • 質問ごとに全員を見る実装では、最大ケースで \(\Theta(NQ)\) 回の確認が必要
  • \(N, Q \le 2 \times 10^5\) なので、最大で \(4 \times 10^{10}\) 回規模になり現実的でない
  • 身長の並び順をうまく使って、まとめて数えられないだろうか？
考えてみる 質問は「\(x\) 以上の人数」。身長の順序を変えても答えが変わらないなら、何をしてから数えると楽になるだろう？

ABC231 C

ソートすると境界が見える

  • 小さい身長側は No
  • ある位置から Yes
  • 最初の Yes が境界

ABC231 C

最初の Yes を bisect_left で探す

判定\(A_i \ge x\)
切り替わりNo ... No, Yes ... Yes
境界\(x\) 以上になる最小の idx

bisect_left(A, x) は、まさに「\(x\) 以上の最初の位置」を返します。

ABC231 C

答えは \(N - \mathrm{idx}\)

  • idx より左は \(x\) 未満
  • idx から右は \(x\) 以上
  • 人数は \(\mathrm{idx}, \mathrm{idx}+1, \ldots, N-1\) の \(N-\mathrm{idx}\) 人
abc231_c_bisect.pyPython
from bisect import bisect_left

N, Q = [int(s) for s in input().split()]
A = [int(s) for s in input().split()]
A.sort()

for _ in range(Q):
    x = int(input())
    idx = bisect_left(A, x)
    ans = N - idx
    print(ans)

ABC388 C - Various Kagamimochi

Various Kagamimochi

AtCoder の問題ページ

問題文

\(N\) 個の餅があります。餅 \(i\) の大きさは \(A_i\) で、配列は昇順に与えられます。

2個の餅を選び、一方を上、一方を下にします。上の餅の大きさが下の餅の半分以下なら、鏡餅として成立します。

少なくとも一方の餅が異なれば、別の鏡餅として数えます。

制約

  • \(2 \le N \le 5 \times 10^5\)
  • \(1 \le A_i \le 10^9\)
  • \(A_i \le A_{i+1}\)

ABC388 C

ペアを直接数える難しさ

  • 2つの餅を全探索すると、調べるペア数は \(\Theta(N^2)\)
  • \(N \le 5 \times 10^5\) なので、最大で約 \(1.25 \times 10^{11}\) 組になり現実的でない
  • すべての組を見ずに、条件を満たす組数だけを数えたい
考えてみる 鏡餅の条件を使って、片方の餅を決めたとき、もう片方をまとめて数えられないだろうか？

ABC388 C

大きい餅 a を固定する

  • 条件: 小さい餅 \(b \le a / 2\)
  • 整数なので b <= a // 2
  • 配列は昇順。A[i] <= a // 2 は左側だけ Yes

ABC388 C

右端の Yes を bisect_right で数える

判定A[i] <= a // 2
切り替わりYes ... Yes, No ... No
境界a // 2 以下の個数

bisect_right(A, a // 2) は「a // 2 以下の要素数」を返します。

ABC388 C

重複・同じサイズ・計算量の確認

  • 同じサイズでも餅が違えば別の組として数える
  • bisect_right は重複分もまとめて数える
  • 各 a について \(O(\log N)\)、合計 \(O(N \log N)\)
abc388_c_bisect.pyPython
from bisect import bisect_right

N = int(input())
A = [int(s) for s in input().split()]
A.sort()

ans = 0
for a in A:
    ans += bisect_right(A, a // 2)

print(ans)

Typical90 001 - Yokan Party

Yokan Party

AtCoder の問題ページ

問題文

長さ \(L\) のようかんがあり、左端から \(A_i\) の位置に \(N\) 個の切れ目があります。

このうち \(K\) 個を選んで切り、\(K+1\) 個のピースに分けます。スコアは、最も短いピースの長さです。

制約

  • \(1 \le K \le N \le 100000\)
  • \(0 < A_1 < \cdots < A_N < L\)

目的

切り方を選んで、スコアの最大値を求めます。

Typical90 001

切り方を直接探すのは難しい

  • \(N\) 個の切れ目から \(K\) 個を選ぶ必要がある
  • 候補になる切り方は組合せ的に増える
  • 求めたいのは「最短ピース長の最大値」
考えてみる 切り方そのものを全部見る代わりに、スコアの値に注目できないだろうか？

Typical90 001

「最小値の最大化」は二分探索

  • 答え候補として、値 x を仮定する
  • 問題を「最短ピース長を \(x\) 以上にできるか？」に変える
  • この Yes / No が一方向に切り替わるなら、境界を探せる
競プロの格言 「最小値の最大化」は二分探索。
探したい境界 Yes になる最大の x が、スコアの最大値になる。

Typical90 001

「最短ピース長を \(x\) 以上にできるか？」

  • 左から見て、長さ \(x\) 以上になったら切る
  • できたピース数が \(K + 1\) 個以上なら Yes
  • この貪欲判定で「できるか」だけを調べる

Typical90 001

\(x\) が小さいほど Yes、大きいほど No

探すのは「できる最大の \(x\)」。だから ok を Yes 側、ng を No 側に置きます。

Typical90 001

貪欲判定 is_valid(x) と ok/ng

  • is_valid(mid) が Yes なら、もっと大きい値を狙う
  • Noなら mid は大きすぎる
  • 最後に残る ok が答え
typical90_001.pyPython
def is_valid(mid):
    piece = 0
    curr = 0
    for pos in cut_pos:
        if pos - curr >= mid:
            piece += 1
            curr = pos
    return piece >= K + 1

ok, ng = 0, L + 1
while ng - ok > 1:
    mid = (ok + ng) // 2
    if is_valid(mid):
        ok = mid
    else:
        ng = mid
