二分探索の基礎へ

Hint

Yes / No の判定方法は？

候補 \(x\) を固定すると、知りたいのは「長さ \(x\) 以上のピースを \(K + 1\) 個作れるか」です。

  • Yes と言うには、条件を満たすピースが \(K + 1\) 個あれば十分
  • No と言うには、どれだけ頑張っても \(K + 1\) 個に届かないと分かればよい
  • では、まず「条件を満たすピースを何個まで作れるか」を数えられないだろうか？

Question

判定を「個数を数える問題」に変える

  • 欲しい判定は「全部のピースを \(x\) 以上にできるか」
  • これは「長さ \(x\) 以上のピースを何個作れるか」と見られる
  • 最大個数が \(K + 1\) 個以上なら、必要なピース数を満たせる
直接考える \(K\) 回切って、全部 \(x\) 以上に分けられるか？
数える形 \(x\) 以上のピースを、できるだけ多く作る
判定 作れた個数が \(K + 1\) 未満なら No
次の疑問 どうすれば「できるだけ多く」作れる？

Greedy

多く作るなら、作れるたびに切る

  • 今のピースが \(x\) 以上なら、条件はもう満たしている
  • さらに右で切ると、その分だけ残りの長さが短くなる
  • だから「最初に切れる場所」で切るのが、後ろのピースに一番やさしい

早く切る貪欲は、後ろに残す余裕を最大にする。

Example

\(L = 34\), \(K = 1\), \(A = [8, 13, 26]\)

1回切って2ピースを作ります。候補 \(x\) を変えると、判定が Yes から No に切り替わります。

\(x = 13\) \(13\) の位置で切ると長さは \(13\) と \(21\)。2ピースとも \(13\) 以上なので Yes。
\(x = 14\) \(26\) の位置まで待つ必要があり、右端までの残りは \(8\)。2ピース作れないので No。

Count

\(K + 1\) 個以上作れたら Yes

  • 判定は「最低でも必要な個数を作れるか」を見る
  • 多く切れそうなら、余分な切れ目を選ばなければよい
  • 右端 \(L\) も最後のピースの終点として数える

Code

is_valid(mid) の読み方

  • curr は、今作っているピースの左端
  • pos - curr >= mid なら、ここで1ピース確定
  • cut_pos に L を入れるので、最後のピースも同じ処理で数えられる

# 0 と L も入れて、端も切れ目として扱う
cut_pos = [0] + A + [L]

def is_valid(mid):
    piece = 0  # 作れたピース数
    curr = 0   # 今のピースの左端
    for pos in cut_pos:
        if pos - curr >= mid:
            piece += 1
            curr = pos
    return piece >= K + 1
